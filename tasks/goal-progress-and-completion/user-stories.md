# Пользовательские истории

Статус: завершено

История
ID: US-01
Название: Просмотр прогресса и статуса цели в списке целей
Источник
- Пользовательский сценарий: пользователь открывает список целей и сразу видит статус и процент прогресса по каждой цели
- Функциональное требование: система должна отображать статус цели в списке целей и в карточке цели
Актор: Пользователь приложения по постановке личных целей
Потребность: видеть статус и процент прогресса по каждой цели в списке
Ценность: быстро понимать текущее состояние всех целей без открытия каждой карточки
Сценарий: пользователь открывает список целей и видит статус и прогресс по каждой цели
Критерии приемки
- для каждой цели в списке отображается процент прогресса
- для каждой цели в списке отображается текущий статус
- если цель завершена, в списке отображается статус `завершена`
Бизнес-правила
- прогресс цели измеряется в процентах от 0 до 100
- статус `завершена` имеет приоритет над промежуточными статусами

- Функциональное требование: система должна отображать статус цели в списке целей и в карточке цели

История
ID: US-03
Название: Обновление прогресса цели
Источник
- Пользовательский сценарий: пользователь обновляет процент прогресса по цели
- Функциональное требование: система должна позволять пользователю изменять прогресс цели
Актор: Пользователь приложения по постановке личных целей
Потребность: обновлять процент выполнения цели
Ценность: фиксировать реальное продвижение по цели и видеть актуальный этап
Сценарий: пользователь открывает цель и изменяет значение прогресса
Критерии приемки
- пользователь может установить прогресс цели в диапазоне от 0 до 100 процентов
- после сохранения новое значение прогресса отображается в карточке цели
- после сохранения новое значение прогресса отображается в списке целей
- при значении меньше 100 цель не переводится автоматически в статус `завершена`
Бизнес-правила
- прогресс цели измеряется в процентах от 0 до 100

История
ID: US-04
Название: Ручное завершение цели
Источник
- Пользовательский сценарий: пользователь вручную помечает цель как завершенную
- Функциональное требование: система должна позволять пользователю вручную завершать цель независимо от текущего процента
Актор: Пользователь приложения по постановке личных целей
Потребность: завершать цель отдельным действием
Ценность: фиксировать достижение цели даже если процент прогресса не был заранее доведен до 100
Сценарий: пользователь в карточке цели выбирает действие завершения и получает статус `завершена`
Критерии приемки
- пользователь может явно пометить цель как завершенную
- после ручного завершения в карточке цели отображается статус `завершена`
- после ручного завершения в списке целей отображается статус `завершена`
Бизнес-правила
- цель может быть завершена вручную пользователем
- статус `завершена` имеет приоритет над промежуточными статусами

История
ID: US-05
Название: Автоматическое завершение цели при 100 процентах
Источник
- Пользовательский сценарий: система автоматически помечает цель как завершенную, когда прогресс достигает 100%
- Функциональное требование: система должна автоматически переводить цель в статус `завершена`, если прогресс установлен в 100%
Актор: Пользователь приложения по постановке личных целей
Потребность: получать автоматическое завершение цели при полном выполнении
Ценность: не делать лишнее действие после достижения полного прогресса
Сценарий: пользователь устанавливает прогресс цели в 100 процентов, и система переводит цель в статус `завершена`
Критерии приемки
- при установке прогресса в 100 процентов цель автоматически получает статус `завершена`
- статус `завершена` отображается в карточке цели
- статус `завершена` отображается в списке целей
Бизнес-правила
- цель может быть завершена автоматически при достижении 100%
- статус `завершена` имеет приоритет над промежуточными статусами
