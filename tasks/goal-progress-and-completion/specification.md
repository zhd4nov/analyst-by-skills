# Спецификация

Статус: завершено

Контекст / Проблема
В приложении по постановке личных целей пользователь может только создать или удалить цель. Из-за отсутствия учета прогресса и статуса непонятно, на каком этапе находится цель, продвигается ли она и была ли она завершена.

Бизнес-цель
Дать пользователю прозрачное понимание состояния каждой цели и возможность фиксировать продвижение и завершение без удаления цели.

Границы объема
- Внутри объема:
  - отображение процента прогресса по цели
  - отображение статуса цели в списке целей и в карточке цели
  - ручное обновление прогресса цели
  - ручное завершение цели
  - автоматическое завершение цели при достижении 100% прогресса
- Вне объема:
  - автоматический расчет прогресса на основе подзадач или чек-листов
  - аналитика по целям и история изменений прогресса

Акторы
- Пользователь приложения по постановке личных целей

Пользовательские сценарии
- пользователь открывает список целей и сразу видит статус и процент прогресса по каждой цели
- пользователь открывает карточку цели и видит текущий статус и процент прогресса
- пользователь обновляет процент прогресса по цели
- пользователь вручную помечает цель как завершенную
- система автоматически помечает цель как завершенную, когда прогресс достигает 100%

Функциональные требования
- система должна хранить и отображать процент прогресса по каждой цели
- система должна отображать статус цели в списке целей и в карточке цели
- система должна позволять пользователю изменять прогресс цели
- система должна позволять пользователю вручную завершать цель независимо от текущего процента
- система должна автоматически переводить цель в статус `завершена`, если прогресс установлен в 100%
- система должна поддерживать статусы `не начата`, `в процессе`, `на паузе`, `просрочена`, `завершена`

Бизнес-правила
- прогресс цели измеряется в процентах от 0 до 100
- цель может быть завершена двумя способами: вручную пользователем или автоматически при достижении 100%
- статус и прогресс должны быть согласованно отображены в списке целей и в карточке цели
- статус `завершена` имеет приоритет над промежуточными статусами

Данные / Сущности
- Цель
- Процент прогресса
- Статус цели

Ограничения
- текущая версия приложения поддерживает только создание и удаление цели, новый функционал должен расширить существующий жизненный цикл цели
- модель прогресса ограничена процентным представлением и не требует этапов или чек-листов в рамках текущего объема
