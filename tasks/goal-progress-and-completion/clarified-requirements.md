# Уточненные требования

Статус: завершено

Акторы
- Пользователь приложения по постановке личных целей

Цели
- Пользователь должен понимать, на каком этапе находится каждая цель
- Пользователь должен иметь возможность фиксировать, продвигается ли цель или уже завершена

Текущий функциональный объем
- отображение текущего прогресса цели
- фиксация изменения прогресса цели
- завершение цели

Ключевые сценарии
- пользователь просматривает список или карточку целей и понимает текущий этап по каждой цели
- пользователь отмечает продвижение по цели
- пользователь завершает цель

Известные ограничения
- в текущей версии приложения цель можно только создать и удалить
- прогресс цели должен отображаться в процентах
- прогресс и статус должны быть видны и в списке целей, и в карточке цели
- цель должна поддерживать ручное и автоматическое завершение
- кроме состояния завершения нужны промежуточные статусы `не начата`, `в процессе`, `на паузе`, `просрочена`
