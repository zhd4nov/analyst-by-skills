# Спецификация

Статус: complete

## Контекст / Проблема
В приложении личных целей пользователь сейчас может только поставить цель и удалить ее. Из-за отсутствия прогресса и завершения пользователь не понимает, на каком этапе находится цель и были ли выполнены действия для ее достижения. Источник: Вход.

## Бизнес-цель
Дать пользователю наблюдаемую картину достижения личных целей: состояние цели, прогресс выполнения для целей со шагами и отдельный список завершенных достижений. Источник: Вход; [Лог уточнений: CL-01]; [Лог уточнений: CL-02].

## Границы объема
- Внутри объема:
  - статусы цели `черновик`, `в процессе`, `завершена`; [Источник: CR-02]
  - процентный прогресс цели со шагами в статусе `в процессе`; [Источник: CR-03, CR-04]
  - шаги-задачи внутри цели, ведущие к выполнению; [Источник: CL-01]
  - подтверждение завершения цели после выполнения всех шагов; [Источник: CR-05, CR-06]
  - добавление новых шагов вместо завершения и пересчет прогресса; [Источник: CR-07]
  - простая цель без шагов с состоянием `выполнена` / `не выполнена`; [Источник: CR-09, CR-10]
  - ручное завершение простой цели пользователем; [Источник: CR-11]
  - перенос завершенной цели из активных целей в достижения; [Источник: CR-08]
  - редактирование цели в процессе, добавление и удаление шагов; [Источник: CR-13]
  - округление процента прогресса до целого числа; [Источник: CR-14]
  - запрет удаления выполненного шага; [Источник: CR-15]
  - переход из черновика в процесс после выполнения первого шага или ручного действия "начать"; [Источник: CR-16, CR-17]
- Вне объема:
  - социальные функции, совместные цели и шаринг достижений.
  - автоматическое определение достижения цели по внешним данным.
  - возврат завершенной цели в активное состояние. [Источник: CR-12]

## Термины и определения
- Цель: пользовательская личная цель, которую можно создать, отслеживать и завершить. Источник: Вход.
- Статус цели: состояние жизненного цикла цели, отдельное от прогресса. [Источник: CR-01]
- Прогресс цели: процент выполнения цели со шагами в статусе `в процессе`, рассчитываемый по выполненным шагам. [Источник: CR-03, CR-04]
- Шаг: задача внутри цели, выполнение которой влияет на процент прогресса цели. [Источник: CL-01]
- Простая цель: цель без шагов, для которой вместо процента используется состояние `выполнена` / `не выполнена`. [Источник: CR-09, CR-10]
- Достижение: завершенная цель, отображаемая вне списка активных целей. [Источник: CR-08]

## Акторы
- Пользователь приложения по постановке личных целей. Источник: Вход.

## Пользовательские сценарии
- Основные:
  - пользователь создает цель и выбирает обычную цель со шагами или простую цель; [Источник: CR-09]
  - пользователь вручную начинает цель действием "начать"; [Источник: CR-17]
  - пользователь просматривает статус цели и прогресс, если цель со шагами находится в процессе; [Источник: CR-01, CR-03, CR-04]
  - пользователь выполняет шаги цели, и процент прогресса пересчитывается и округляется до целого числа; [Источник: CR-04, CR-14]
  - пользователь выполняет первый шаг цели, и цель переходит из `черновик` в `в процессе`; [Источник: CR-16]
  - пользователь редактирует цель в процессе, добавляет шаги или удаляет невыполненные шаги; [Источник: CR-13, CR-15]
  - после выполнения всех шагов пользователь подтверждает завершение цели или добавляет новые шаги; [Источник: CR-05, CR-06, CR-07]
  - пользователь вручную завершает простую цель; [Источник: CR-11]
  - пользователь видит завершенную цель в списке достижений, а не среди активных целей. [Источник: CR-08]
- Альтернативные:
  - если после выполнения всех шагов пользователь добавляет новые шаги, цель остается в процессе, а процент прогресса пересчитывается. [Источник: CR-07]
- Исключительные:
  - завершенную цель нельзя вернуть назад в активное состояние. [Источник: CR-12]

## Функциональные требования
- Система должна хранить и показывать статус цели: `черновик`, `в процессе`, `завершена`. [Источник: CR-02]
- Система должна отличать статус цели от прогресса цели. [Источник: CR-01]
- Для цели со шагами в статусе `в процессе` система должна показывать процент прогресса. [Источник: CR-03, CR-04]
- Система должна рассчитывать процент прогресса цели со шагами на основе выполненных шагов. [Источник: CR-04]
- Система должна отображать процент прогресса целым числом с округлением расчетного значения. [Источник: CR-14]
- Система должна позволять пользователю добавлять шаги и удалять невыполненные шаги у цели в статусе `в процессе`. [Источник: CR-13, CR-15]
- Система должна пересчитывать процент прогресса после изменения состава или выполнения шагов. [Источник: CR-04, CR-07, CR-13]
- Система должна переводить цель из `черновик` в `в процессе` после выполнения первого шага. [Источник: CR-16]
- Система должна позволять пользователю вручную перевести цель из `черновик` в `в процессе` действием "начать". [Источник: CR-17]
- Когда все шаги выполнены, система должна запросить у пользователя подтверждение достижения цели или предложить добавить еще шаги. [Источник: CR-05]
- Если пользователь подтверждает достижение цели, система должна перевести цель в статус `завершена`. [Источник: CR-06]
- Если пользователь добавляет шаги вместо подтверждения завершения, система должна оставить цель в процессе и пересчитать процент прогресса. [Источник: CR-07]
- Система должна позволять пользователю отметить цель как простую при создании. [Источник: CR-09]
- Для простой цели система должна показывать состояние `выполнена` / `не выполнена` вместо процента. [Источник: CR-10]
- Простая цель должна завершаться только вручную пользователем. [Источник: CR-11]
- Завершенная цель должна отображаться в списке достижений и не отображаться в списке активных целей. [Источник: CR-08]

## Валидационные правила
- Процент прогресса применяется только к цели со шагами в статусе `в процессе`. [Источник: CR-03]
- Процент прогресса всегда отображается целым числом; расчетное значение округляется. [Источник: CR-14]
- Простая цель не должна иметь процент прогресса. [Источник: CR-10]
- Выполненный шаг нельзя удалить. [Источник: CR-15]
- Завершенная цель не должна переводиться назад в `черновик` или `в процессе`. [Источник: CR-12]
- Завершение цели со шагами через автоматический переход запрещено без подтверждения пользователя. [Источник: CR-05, CR-06]
- Если пользователь добавил шаги на этапе подтверждения завершения, завершение не выполняется, а прогресс пересчитывается. [Источник: CR-07]

## Бизнес-правила
- [Источник: CR-01] Статус и прогресс цели не должны подменять друг друга.
- [Источник: CR-03] Процентный прогресс отображается только для цели со шагами в статусе `в процессе`.
- [Источник: CR-04] Процент прогресса зависит от выполненных шагов внутри цели.
- [Источник: CR-14] Процент прогресса всегда отображается целым числом и округляется.
- [Источник: CR-07] Добавление шагов вместо подтверждения завершения оставляет цель в процессе и меняет расчет прогресса.
- [Источник: CR-10] Простая цель имеет бинарное состояние выполнения вместо процента.
- [Источник: CR-11] Простая цель завершается только ручным действием пользователя.
- [Источник: CR-13] Цель в процессе можно редактировать, добавлять шаги и удалять невыполненные шаги.
- [Источник: CR-15] Выполненный шаг удалить нельзя.

## Модель состояний / Переходы
- Начальное состояние: `черновик`. [Источник: CR-02]
- Состояния:
  - `черновик`; [Источник: CR-02]
  - `в процессе`; [Источник: CR-02]
  - `завершена`; [Источник: CR-02]
- Переходы:
  - `черновик` -> `в процессе`
    - Инициатор: пользователь.
    - Событие / условие: пользователь выполняет первый шаг.
    - Источник: CR-16.
  - `черновик` -> `в процессе`
    - Инициатор: пользователь.
    - Событие / условие: пользователь вручную выбирает действие "начать".
    - Источник: CR-17.
  - `в процессе` -> `завершена` для цели со шагами
    - Инициатор: пользователь после запроса системы.
    - Событие / условие: все шаги выполнены, система спрашивает, достигнута ли цель, пользователь подтверждает достижение.
    - Источник: CR-05, CR-06.
  - `в процессе` -> `в процессе` для цели со шагами
    - Инициатор: пользователь.
    - Событие / условие: после выполнения всех шагов пользователь добавляет еще шаги вместо подтверждения завершения; система пересчитывает прогресс.
    - Источник: CR-07.
  - `в процессе` -> `завершена` для простой цели
    - Инициатор: пользователь.
    - Событие / условие: пользователь вручную завершает простую цель.
    - Источник: CR-11.
  - `завершена` -> любое другое состояние
    - Инициатор: не допускается.
    - Событие / условие: переход запрещен, статус терминальный.
    - Источник: CR-12.

## Данные / Сущности
- Цель:
  - статус: `черновик`, `в процессе`, `завершена`; [Источник: CR-02]
  - признак простой цели; [Источник: CR-09]
  - состояние простой цели: `выполнена` / `не выполнена`, если цель простая; [Источник: CR-10]
  - процент прогресса, если цель со шагами находится в процессе; [Источник: CR-03, CR-04]
- Шаг:
  - принадлежность к цели; [Источник: CL-01]
  - признак выполнения; [Источник: CL-01]
- Список активных целей:
  - содержит незавершенные цели. [Источник: CR-08]
- Список достижений:
  - содержит завершенные цели. [Источник: CR-08]

## Ограничения
- [Источник: CR-12] Завершенная цель является терминальной и не может быть возвращена назад.
- [Источник: CR-13, CR-15] Редактирование, добавление шагов и удаление невыполненных шагов разрешены только пока цель в процессе; выполненный шаг удалить нельзя.
- [Источник: Вход] В текущей версии приложения цель можно только поставить и удалить.
