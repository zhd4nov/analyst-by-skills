# Спецификация

Статус: завершено

Контекст / Проблема
В приложении по постановке личных целей пользователь может только создать или удалить цель. Из-за этого нельзя понять, продвигается ли цель, на каком этапе она находится и была ли она завершена. Удаление цели не заменяет фиксацию результата, поэтому пользователь теряет прозрачность состояния работы по целям.

Бизнес-цель
Дать пользователю прозрачный способ видеть текущее состояние цели, фиксировать прогресс по ней и завершать цель без удаления.

Границы объема
- Внутри объема:
  - отображение процента прогресса по цели
  - отображение шагов прогресса по цели
  - отображение статуса цели в списке целей и в карточке цели
  - ручное обновление процента прогресса
  - задание шагов при создании цели
  - отметка шагов как выполненных
  - ручное завершение цели
- Вне объема:
  - автоматический расчет прогресса по внешним данным
  - повторное открытие завершенной цели
  - редактирование шагов после создания цели
  - аналитика по истории изменений прогресса

Акторы
- Пользователь приложения по постановке личных целей

Пользовательские сценарии
- пользователь открывает список целей и видит статус, процент прогресса и состояние шагов по каждой цели
- пользователь открывает карточку цели и видит детальное состояние прогресса и текущий статус
- пользователь создает цель и задает шаги для ее выполнения
- пользователь обновляет процент прогресса цели
- пользователь отмечает выполненность шагов цели
- пользователь завершает цель и фиксирует ее как окончательно выполненную

Функциональные требования
- система должна хранить и отображать прогресс цели одновременно в двух формах: процент и шаги
- система должна показывать статус цели в списке целей и в карточке цели
- система должна позволять пользователю изменять процент прогресса цели
- система должна позволять пользователю задавать шаги цели при ее создании
- система должна позволять пользователю отмечать шаги цели как выполненные
- система не должна позволять редактировать набор шагов после создания цели
- система должна позволять пользователю переводить цель в статус `завершена`
- система не должна позволять возвращать цель из статуса `завершена` в активные статусы
- система должна поддерживать статусы `не начата`, `в процессе`, `на паузе`, `завершена`

Бизнес-правила
- [Источник: Вход] пользователь должен понимать, сделал ли он что-то по цели или нет
- [Источник: Уточненные требования] прогресс цели должен вестись одновременно в процентах и в шагах
- [Источник: Уточненные требования] все заданные шаги цели в совокупности составляют 100% прогресса по шагам
- [Источник: Уточненные требования] проценты не пересчитываются в шаги
- [Источник: Уточненные требования] шаги цели задаются только при создании цели и не редактируются после создания
- [Источник: Допущение] в первой версии обязательные рабочие статусы цели: `не начата`, `в процессе`, `на паузе`, `завершена`
- [Источник: Уточненные требования] после перевода цели в статус `завершена` обратный переход не допускается
- [Источник: Уточненные требования] переход в статус `в процессе` выполняется вручную пользователем или автоматически при отметке хотя бы одного шага как выполненного
- [Источник: Уточненные требования] переход в статус `на паузе` выполняется только вручную пользователем

Модель состояний / Переходы
- Начальное состояние: `не начата`
- Состояния:
  - `не начата`
  - `в процессе`
  - `на паузе`
  - `завершена`
- Переходы:
  - `не начата` -> `в процессе`
    - Инициатор: пользователь или система
    - Событие / условие: пользователь вручную переводит цель в активную работу либо пользователь отмечает хотя бы один шаг как выполненный
    - Источник: Уточненные требования
  - `в процессе` -> `на паузе`
    - Инициатор: пользователь
    - Событие / условие: пользователь вручную ставит цель на паузу
    - Источник: Уточненные требования
  - `на паузе` -> `в процессе`
    - Инициатор: пользователь
    - Событие / условие: пользователь вручную возобновляет работу по цели
    - Источник: Уточненные требования
  - `не начата` -> `завершена`
    - Инициатор: пользователь
    - Событие / условие: пользователь вручную завершает цель без промежуточной фиксации прогресса
    - Источник: Уточненные требования
  - `в процессе` -> `завершена`
    - Инициатор: пользователь
    - Событие / условие: пользователь вручную завершает цель
    - Источник: Уточненные требования
  - `на паузе` -> `завершена`
    - Инициатор: пользователь
    - Событие / условие: пользователь вручную завершает цель
    - Источник: Уточненные требования

Данные / Сущности
- Цель
- Статус цели
- Процент прогресса
- Шаг цели
- Признак выполнения шага

Ограничения
- [Источник: Вход] в текущем состоянии приложения цель можно только создать или удалить
- [Источник: Уточненные требования] в первой версии прогресс должен поддерживаться одновременно в виде процента и шагов
- [Источник: Уточненные требования] шаги цели задаются только при создании цели и не редактируются после создания
- [Источник: Уточненные требования] после завершения цель считается окончательно завершенной и не может быть возвращена в активную работу

Допущения
- в первой версии обязательные рабочие статусы цели: `не начата`, `в процессе`, `на паузе`, `завершена`
