# Уточненные требования

Статус: завершено

Акторы
- Пользователь приложения по постановке личных целей

Цели
- Понимать, на каком этапе находится каждая цель
- Фиксировать, есть ли продвижение по цели
- Завершать цель без ее удаления

Текущий функциональный объем
- Добавить учет прогресса по цели
- Показать текущее состояние цели пользователю
- Добавить явное завершение цели

Ключевые сценарии
- пользователь просматривает список или карточку целей и видит текущий этап / состояние цели
- пользователь фиксирует, что по цели есть продвижение
- пользователь помечает цель как завершенную

Известные ограничения
- В текущем состоянии приложения цель можно только создать или удалить
- В первой версии прогресс должен поддерживаться одновременно в виде процента и шагов
- После перевода цели в статус `завершена` обратный переход не допускается
- Все заданные шаги цели в совокупности составляют 100% прогресса
- Проценты не пересчитываются в шаги
- Шаги цели задаются только при создании цели и не редактируются после создания
- Переход в статус `на паузе` выполняется только вручную пользователем

Допущения
- в первой версии обязательные рабочие статусы цели: `не начата`, `в процессе`, `на паузе`, `завершена`

Открытые вопросы
- нет
