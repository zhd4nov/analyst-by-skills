# Пользовательские истории

Статус: завершено

История
ID: US-01
Название: Просмотр состояния цели в списке и карточке
Источник
- Пользовательский сценарий: пользователь открывает список целей и видит статус, процент прогресса и состояние шагов по каждой цели
- Пользовательский сценарий: пользователь открывает карточку цели и видит детальное состояние прогресса и текущий статус
- Функциональное требование: система должна показывать статус цели в списке целей и в карточке цели
Актор: Пользователь приложения по постановке личных целей
Потребность: видеть текущее состояние цели, процент прогресса и шаги
Ценность: понимать, на каком этапе находится цель и есть ли продвижение
Сценарий: пользователь открывает список или карточку цели и видит статус, процент прогресса и состояние шагов
Критерии приемки
- в списке целей для каждой цели отображается текущий статус
- в списке целей для каждой цели отображается текущий процент прогресса
- в карточке цели отображаются статус, процент прогресса и шаги цели
- если по цели нет выполненных шагов и прогресс не обновлялся, пользователь все равно видит ее текущее состояние
Бизнес-правила
- прогресс цели должен вестись одновременно в процентах и в шагах
- обязательные рабочие статусы цели: `не начата`, `в процессе`, `на паузе`, `завершена`
- все заданные шаги цели в совокупности составляют 100% прогресса по шагам
Зависимости
- US-02
- US-03
- US-04
- US-05

История
ID: US-02
Название: Создание цели с шагами
Источник
- Пользовательский сценарий: пользователь создает цель и задает шаги для ее выполнения
- Функциональное требование: система должна позволять пользователю задавать шаги цели при ее создании
- Функциональное требование: система не должна позволять редактировать набор шагов после создания цели
Актор: Пользователь приложения по постановке личных целей
Потребность: задать шаги цели в момент ее создания
Ценность: сразу зафиксировать состав работы и затем отмечать реальное продвижение
Сценарий: пользователь создает цель и указывает набор шагов, который затем используется для отслеживания выполнения
Критерии приемки
- при создании цели пользователь может добавить шаги цели
- сохраненные шаги становятся частью данных цели
- после создания цели пользователь не может изменить, удалить или добавить шаги
Бизнес-правила
- шаги цели задаются только при создании цели и не редактируются после создания
Зависимости
- нет

История
ID: US-03
Название: Обновление процентного прогресса цели
Источник
- Пользовательский сценарий: пользователь обновляет процент прогресса цели
- Функциональное требование: система должна позволять пользователю изменять процент прогресса цели
Актор: Пользователь приложения по постановке личных целей
Потребность: обновлять процент прогресса по цели
Ценность: фиксировать, насколько далеко цель продвинулась
Сценарий: пользователь открывает цель и изменяет процент ее прогресса
Критерии приемки
- пользователь может задать или изменить процент прогресса цели
- сохраненное значение процента отображается в списке целей и в карточке цели
- значение процента хранится как часть данных цели
Бизнес-правила
- прогресс цели должен вестись одновременно в процентах и в шагах
- проценты не пересчитываются в шаги

История
ID: US-04
Название: Отметка выполнения шагов цели
Источник
- Пользовательский сценарий: пользователь отмечает выполненность шагов цели
- Функциональное требование: система должна позволять пользователю отмечать шаги цели как выполненные
Актор: Пользователь приложения по постановке личных целей
Потребность: фиксировать выполнение отдельных шагов по цели
Ценность: понимать, что именно уже сделано по цели
Сценарий: пользователь открывает цель и отмечает отдельные шаги как выполненные
Критерии приемки
- пользователь видит список шагов цели
- пользователь может отметить шаг цели как выполненный
- состояние выполнения шага сохраняется и отображается при повторном открытии цели
- если пользователь отмечает хотя бы один шаг как выполненный, цель переходит в статус `в процессе`
Бизнес-правила
- прогресс цели должен вестись одновременно в процентах и в шагах
- все заданные шаги цели в совокупности составляют 100% прогресса по шагам
- шаги цели задаются только при создании цели и не редактируются после создания
- переход в статус `в процессе` выполняется автоматически при отметке хотя бы одного шага как выполненного
Зависимости
- US-02

История
ID: US-05
Название: Окончательное завершение цели
Источник
- Пользовательский сценарий: пользователь завершает цель и фиксирует ее как окончательно выполненную
- Функциональное требование: система должна позволять пользователю переводить цель в статус `завершена`
- Функциональное требование: система не должна позволять возвращать цель из статуса `завершена` в активные статусы
Актор: Пользователь приложения по постановке личных целей
Потребность: явно завершать цель без ее удаления
Ценность: сохранять результат и понимать, что работа по цели закончена
Сценарий: пользователь вручную переводит цель в статус `завершена`
Критерии приемки
- пользователь может перевести цель в статус `завершена`
- после завершения статус цели отображается как `завершена` в списке и карточке
- после завершения цель нельзя вернуть в статусы `не начата`, `в процессе` или `на паузе`
Бизнес-правила
- после перевода цели в статус `завершена` обратный переход не допускается

- нет
