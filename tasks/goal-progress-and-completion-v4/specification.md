# Спецификация

Статус: завершено

Контекст / Проблема
В приложении по постановке личных целей пользователь может только создать или удалить цель. Из-за этого он не понимает, на каком этапе находится цель, есть ли по ней реальное продвижение и была ли она достигнута. Удаление цели не отличает отказ от достижения и не дает прозрачной картины по активным и завершенным целям.

Бизнес-цель
Дать пользователю прозрачный способ видеть состояние цели, фиксировать прогресс по ней и завершать цель как достижение, а не как удаление.

Границы объема
- Внутри объема:
  - отображение статуса цели
  - отображение прогресса цели в процентах
  - поддержка целей со шагами и без шагов
  - отметка выполнения шагов с автоматическим пересчетом прогресса
  - автоматический переход цели из `план` в `в процессе` при выполнении первого шага
  - ручное изменение статусов `план`, `в процессе`, `пауза` в допустимых пределах
  - завершение цели со шагами после выполнения всех шагов
  - завершение цели без шагов явной отметкой выполнения
  - перенос завершенной цели в достижения
  - исключение завершенной цели из списка активных целей
- Вне объема:
  - ручной ввод процента прогресса
  - ручная установка статуса `завершена` без выполнения условий завершения
  - возврат завершенной цели в активные статусы
  - аналитика по истории изменения прогресса
  - отдельный архив удаленных целей

Термины и определения
- Цель: пользовательская сущность личного намерения, которую можно планировать, выполнять, ставить на паузу и завершать
- Шаг: элемент плана цели, который задается пользователем и может быть отмечен как выполненный
- Прогресс цели: вычисляемый процент выполнения цели
- Активные цели: цели, которые еще не завершены и отображаются в основном списке
- Достижения: завершенные цели, которые больше не показываются в списке активных целей
- Удаление цели: отказ пользователя от цели без фиксации достижения

Акторы
- Пользователь приложения по постановке личных целей

Пользовательские сценарии
- Основные:
  - пользователь открывает список активных целей и видит статус и прогресс каждой цели
  - пользователь создает цель со шагами и отмечает шаги выполненными
  - пользователь выполняет первый шаг и цель автоматически переходит из `план` в `в процессе`
  - пользователь вручную ставит цель на `паузу` и затем возвращает ее в `в процессе`
  - пользователь завершает цель со шагами после выполнения всех шагов
  - пользователь завершает цель без шагов явной отметкой выполнения
  - пользователь открывает достижения и видит там завершенные цели
  - пользователь удаляет цель и тем самым отказывается от нее
- Альтернативные:
  - пользователь вручную переводит цель из `план` в `в процессе`, не дожидаясь выполнения первого шага
- Исключительные:
  - пользователь пытается вручную установить статус `завершена` до выполнения всех шагов
  - пользователь пытается вручную изменить процент прогресса
  - пользователь пытается вернуть завершенную цель в активный список

Функциональные требования
- система должна показывать статус цели в списке активных целей и в карточке цели
- система должна показывать прогресс цели в процентах
- система должна позволять пользователю создавать цель со шагами
- система должна позволять пользователю создавать цель без шагов
- система должна автоматически рассчитывать прогресс как отношение выполненных шагов к общему количеству шагов, если шаги заданы
- система должна показывать прогресс `0%` для цели без шагов до явной отметки выполнения
- система должна показывать прогресс `100%` для цели без шагов после явной отметки выполнения
- система должна позволять пользователю отмечать шаги выполненными
- система должна автоматически пересчитывать прогресс после изменения состояния шага
- система должна автоматически переводить цель из `план` в `в процессе` при выполнении первого шага
- система должна позволять пользователю вручную переводить цель между статусами `план`, `в процессе` и `пауза`
- система не должна позволять пользователю вручную менять процент прогресса
- система не должна позволять вручную установить статус `завершена`, пока цель со шагами не выполнена полностью
- система должна позволять завершить цель со шагами только после выполнения всех шагов
- система должна позволять пользователю явно отметить выполненной цель без шагов
- система должна переносить завершенную цель в достижения
- система не должна показывать завершенную цель в списке активных целей
- система должна отличать завершение цели от удаления цели

Бизнес-правила
- [Источник: Лог уточнений CL-01] прогресс цели со шагами считается в процентах от выполненных шагов
- [Источник: Лог уточнений CL-01] если у цели нет шагов, прогресс цели дискретный: `0%` или `100%`
- [Источник: Лог уточнений CL-02] допустимые статусы цели: `план`, `в процессе`, `пауза`, `завершена`
- [Источник: Лог уточнений CL-03] завершение цели означает отметку выполнения, перенос в достижения и удаление из списка активных целей
- [Источник: Лог уточнений CL-04] пользователь может вручную менять только статусы `план`, `в процессе`, `пауза`
- [Источник: Лог уточнений CL-04] статус `завершена` достигается только при выполнении всех шагов либо при явном выполнении цели без шагов
- [Источник: Лог уточнений CL-04] выполнение первого шага автоматически переводит цель из `план` в `в процессе`

Модель состояний / Переходы
- Начальное состояние: `план`
- Состояния:
  - `план`
  - `в процессе`
  - `пауза`
  - `завершена`
- Переходы:
  - `план` -> `в процессе`
    - Инициатор: пользователь или система
    - Событие / условие: пользователь вручную начинает работу по цели либо выполняет первый шаг
    - Источник: Лог уточнений CL-04
  - `в процессе` -> `пауза`
    - Инициатор: пользователь
    - Событие / условие: пользователь вручную ставит цель на паузу
    - Источник: Лог уточнений CL-04
  - `пауза` -> `в процессе`
    - Инициатор: пользователь
    - Событие / условие: пользователь вручную возобновляет работу по цели
    - Источник: Лог уточнений CL-04
  - `в процессе` -> `завершена`
    - Инициатор: система и пользователь
    - Событие / условие: все шаги выполнены и цель фиксируется как выполненная
    - Источник: Лог уточнений CL-04
  - `план` -> `завершена`
    - Инициатор: пользователь
    - Событие / условие: у цели нет шагов, пользователь явно отмечает цель выполненной
    - Источник: Лог уточнений CL-04
  - `пауза` -> `завершена`
    - Инициатор: пользователь
    - Событие / условие: у цели нет шагов, пользователь явно отмечает цель выполненной
    - Источник: Лог уточнений CL-04

Валидационные правила
- процент прогресса должен быть производным от числа выполненных шагов, если шаги существуют
- для цели без шагов допустимы только значения прогресса `0%` и `100%`
- завершенная цель не должна отображаться в списке активных целей
- завершенную цель нельзя вернуть в статусы `план`, `в процессе` или `пауза`
- удаление цели не должно помещать цель в достижения

Данные / Сущности
- Цель
- Статус цели
- Шаг цели
- Признак выполнения шага
- Процент прогресса
- Список активных целей
- Раздел достижений
- Признак удаления цели
