# Уточненные требования

Статус: завершено

Акторы
- Пользователь приложения по постановке личных целей

Цели
- Понимать, на каком этапе находится каждая цель
- Видеть, есть ли реальное продвижение по цели
- Завершать цель без удаления и отличать достижение от отказа

Текущий функциональный объем
- Добавить отображение прогресса цели в процентах
- Добавить статусы цели `план`, `в процессе`, `пауза`, `завершена`
- Добавить выполнение шагов цели с автоматическим пересчетом прогресса
- Добавить завершение цели со шагами и без шагов
- Переносить завершенные цели в достижения и убирать их из списка активных целей

Ключевые сценарии
- пользователь открывает список активных целей и видит статус и процент прогресса каждой цели
- пользователь создает цель со шагами и отмечает шаги выполненными
- пользователь видит автоматический пересчет прогресса после выполнения шагов
- пользователь вручную переводит цель между статусами `план`, `в процессе` и `пауза`
- пользователь завершает цель после выполнения всех шагов
- пользователь завершает цель без шагов явной отметкой выполнения
- пользователь открывает достижения и видит там завершенные цели

Известные ограничения
- [Источник: Вход] в текущем состоянии приложения цель можно только создать или удалить
- [Источник: Лог уточнений CL-01] прогресс считается в процентах от выполненных шагов
- [Источник: Лог уточнений CL-01] если у цели нет шагов, прогресс дискретный: `0%` до выполнения и `100%` после выполнения
- [Источник: Лог уточнений CL-02] допустимые статусы цели: `план`, `в процессе`, `пауза`, `завершена`
- [Источник: Лог уточнений CL-03] завершенная цель переносится в достижения и исчезает из списка активных целей
- [Источник: Лог уточнений CL-04] прогресс нельзя менять вручную, он обновляется по выполнению шагов
- [Источник: Лог уточнений CL-04] статусы `план`, `в процессе`, `пауза` могут меняться вручную пользователем
- [Источник: Лог уточнений CL-04] статус `завершена` нельзя установить вручную для цели со шагами до выполнения всех шагов
- [Источник: Лог уточнений CL-04] при выполнении первого шага цель из `план` автоматически переходит в `в процессе`
- [Источник: Лог уточнений CL-04] цель без шагов может быть завершена только явной отметкой выполнения пользователем
