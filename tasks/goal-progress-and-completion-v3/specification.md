# Спецификация

Статус: завершено

Контекст / Проблема
В приложении по постановке личных целей пользователь может только создать или удалить цель. Из-за этого нельзя понять, есть ли реальное продвижение по цели, на каком этапе она находится и была ли цель достигнута или от нее отказались. Удаление цели смешивает отказ и достижение, поэтому пользователь теряет историю результата.

Бизнес-цель
Дать пользователю прозрачный способ видеть состояние цели, фиксировать реальный прогресс и отличать достижение цели от отказа от нее.

Границы объема
- Внутри объема:
  - отображение статуса цели
  - отображение процента прогресса по цели
  - создание цели с шагами или без шагов
  - автоматический расчет прогресса по выполненным шагам
  - дискретное завершение цели без шагов
  - отметка выполнения отдельных шагов
  - автоматическое завершение цели при выполнении всех шагов после подтверждения пользователя
  - ручной перевод цели в `работаем`
  - ручная постановка цели на `паузу`
  - отображение завершенной цели в разделе достижений
  - удаление цели как отдельное действие отказа
- Вне объема:
  - ручной ввод процента прогресса
  - возврат цели из `завершена` в рабочие статусы
  - возврат цели в `черновик`
  - редактирование шагов в любой момент, кроме случая отказа завершить цель после выполнения последнего шага
  - аналитика по истории изменений прогресса

Термины и определения
- Цель: пользовательская сущность личного намерения, которая может быть в работе, на паузе, завершена или удалена
- Шаг: элемент плана цели, который задается пользователем и может быть отмечен как выполненный
- Прогресс цели: вычисляемый показатель выполнения цели в процентах
- Достижение: завершенная цель, которая сохраняется в отдельном разделе результатов пользователя
- Удаление цели: отказ пользователя от цели без фиксации достижения

Акторы
- Пользователь приложения по постановке личных целей

Пользовательские сценарии
- Основные:
  - пользователь открывает список целей и видит статус и прогресс каждой цели
  - пользователь создает цель с шагами и дальше отмечает выполненные шаги
  - пользователь создает цель без шагов и отмечает ее выполненной целиком
  - пользователь завершает последний шаг и подтверждает завершение цели
  - пользователь открывает раздел достижений и видит завершенную цель
  - пользователь вручную переводит цель из `черновик` в `работаем`
  - пользователь вручную ставит цель на `паузу` и затем возобновляет работу
  - пользователь удаляет цель и тем самым отказывается от нее
- Альтернативные:
  - пользователь завершает последний шаг, но отказывается завершать цель и добавляет новые шаги
- Исключительные:
  - пользователь пытается вернуть цель из `завершена` в рабочий статус
  - пользователь пытается вернуть цель из `работаем` или `пауза` в `черновик`

Функциональные требования
- система должна показывать статус цели в списке целей и в карточке цели
- система должна показывать вычисляемый процент прогресса по цели
- система должна позволять пользователю создать цель с шагами
- система должна позволять пользователю создать цель без шагов
- система должна автоматически рассчитывать прогресс цели по количеству выполненных шагов, если шаги заданы
- система должна показывать прогресс `0%` для цели без шагов, пока она не отмечена выполненной
- система должна показывать прогресс `100%` для цели без шагов после явной отметки выполнения пользователем
- система должна позволять пользователю отмечать шаги цели как выполненные
- система должна автоматически переводить цель из `черновик` в `работаем` при выполнении первого шага
- система должна позволять пользователю вручную переводить цель из `черновик` в `работаем`
- система должна позволять пользователю вручную переводить цель из `работаем` в `пауза`
- система должна позволять пользователю вручную переводить цель из `пауза` в `работаем`
- система должна предлагать пользователю завершить цель после выполнения последнего шага
- система должна автоматически переводить цель в `завершена` после подтверждения пользователем при выполненных всех шагах
- система должна позволять пользователю добавить новые шаги, если он отказался завершить цель после выполнения последнего шага
- система должна пересчитывать прогресс цели после добавления новых шагов в этом сценарии
- система должна позволять пользователю явно отметить выполненной цель без шагов
- система должна показывать завершенные цели в разделе достижений
- система должна запрещать повторное открытие цели из статуса `завершена`
- система должна запрещать возврат цели в статус `черновик` после выхода из него
- система должна сохранять удаление цели как отдельное действие отказа, отличное от завершения

Валидационные правила
- процент прогресса недоступен для ручного редактирования пользователем
- если у цели есть шаги, прогресс рассчитывается только на основании выполненных шагов
- если у цели нет шагов, допустимые значения прогресса только `0%` и `100%`
- перевод в `завершена` допустим только после выполнения всех шагов и подтверждения пользователя либо после явной отметки выполнения цели без шагов
- возврат из `завершена` в любой другой статус недопустим
- возврат в `черновик` после перехода в `работаем` или `пауза` недопустим

Бизнес-правила
- [Источник: Вход] пользователь должен понимать, сделал ли он что-то по цели или нет
- [Источник: Уточненные требования] прогресс цели рассчитывается от шагов, если шаги были добавлены при создании цели
- [Источник: Уточненные требования] пользователь не задает процент прогресса вручную
- [Источник: Уточненные требования] если шаги отсутствуют, прогресс цели дискретный: `0%` или `100%`
- [Источник: Уточненные требования] первый выполненный шаг автоматически переводит цель из `черновик` в `работаем`
- [Источник: Уточненные требования] пользователь может вручную перевести цель из `черновик` в `работаем`
- [Источник: Уточненные требования] статус `пауза` устанавливается только вручную пользователем
- [Источник: Уточненные требования] возврат цели в `черновик` не допускается
- [Источник: Уточненные требования] цель со шагами завершается автоматически после выполнения всех шагов и подтверждения пользователя
- [Источник: Уточненные требования] если пользователь отказался завершить цель после выполнения последнего шага, он должен иметь возможность добавить новые шаги, а прогресс цели должен быть пересчитан
- [Источник: Уточненные требования] цель без шагов завершается после явной отметки выполнения пользователем
- [Источник: Уточненные требования] статус `завершена` терминальный для сущности цели
- [Источник: Уточненные требования] завершенная цель должна отображаться в разделе достижений
- [Источник: Уточненные требования] удаление цели означает отказ от цели, а завершение означает достижение

Модель состояний / Переходы
- Начальное состояние: `черновик`
- Состояния:
  - `черновик`
  - `работаем`
  - `пауза`
  - `завершена`
- Переходы:
  - `черновик` -> `работаем`
    - Инициатор: пользователь или система
    - Событие / условие: пользователь вручную начинает работу по цели либо выполняет первый шаг
    - Источник: Уточненные требования
  - `работаем` -> `пауза`
    - Инициатор: пользователь
    - Событие / условие: пользователь вручную ставит цель на паузу
    - Источник: Уточненные требования
  - `пауза` -> `работаем`
    - Инициатор: пользователь
    - Событие / условие: пользователь вручную возобновляет работу по цели
    - Источник: Уточненные требования
  - `работаем` -> `завершена`
    - Инициатор: пользователь и система
    - Событие / условие: выполнены все шаги, система спрашивает подтверждение завершения, пользователь подтверждает завершение
    - Источник: Уточненные требования
  - `черновик` -> `завершена`
    - Инициатор: пользователь
    - Событие / условие: у цели нет шагов, пользователь явно отмечает цель выполненной
    - Источник: Уточненные требования
  - `пауза` -> `завершена`
    - Инициатор: пользователь
    - Событие / условие: у цели нет шагов и пользователь явно отмечает цель выполненной после возобновления логики завершения без возврата в работу
    - Источник: Уточненные требования

Данные / Сущности
- Цель
- Статус цели
- Шаг цели
- Признак выполнения шага
- Вычисляемый процент прогресса
- Раздел достижений
- Признак удаления цели
