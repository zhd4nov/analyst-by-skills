# Канонические правила

Статус: завершено

Правило
ID: CR-01
Тип: термин
Формулировка: Цель может находиться в одном из статусов `план`, `в процессе`, `пауза`, `завершена`.
Источник: Лог уточнений: CL-02
Использовать дальше:
- Спецификация -> Термины и определения
- Спецификация -> Модель состояний / Переходы
- Пользовательские истории -> US-01, US-04, US-05, US-06

Правило
ID: CR-02
Тип: бизнес-правило
Формулировка: Прогресс цели со шагами считается в процентах от выполненных шагов.
Источник: Лог уточнений: CL-01
Использовать дальше:
- Спецификация -> Функциональные требования
- Спецификация -> Бизнес-правила
- Пользовательские истории -> US-01, US-02, US-03

Правило
ID: CR-03
Тип: валидация
Формулировка: Если у цели нет шагов, прогресс может принимать только значения `0%` и `100%`.
Источник: Лог уточнений: CL-01
Использовать дальше:
- Спецификация -> Функциональные требования
- Спецификация -> Валидационные правила
- Пользовательские истории -> US-01, US-06

Правило
ID: CR-04
Тип: видимость
Формулировка: Завершенная цель переносится в достижения и не отображается в списке активных целей.
Источник: Лог уточнений: CL-03
Использовать дальше:
- Спецификация -> Функциональные требования
- Спецификация -> Ограничения
- Пользовательские истории -> US-05, US-06

Правило
ID: CR-05
Тип: переход состояния
Формулировка: При выполнении первого шага цель автоматически переходит из `план` в `в процессе`.
Источник: Лог уточнений: CL-04
Использовать дальше:
- Спецификация -> Модель состояний / Переходы
- Пользовательские истории -> US-03

Правило
ID: CR-06
Тип: бизнес-правило
Формулировка: Пользователь может вручную менять только статусы `план`, `в процессе` и `пауза`.
Источник: Лог уточнений: CL-04
Использовать дальше:
- Спецификация -> Бизнес-правила
- Спецификация -> Модель состояний / Переходы
- Пользовательские истории -> US-04

Правило
ID: CR-07
Тип: валидация
Формулировка: Пользователь не может вручную менять процент прогресса цели.
Источник: Лог уточнений: CL-04
Использовать дальше:
- Спецификация -> Функциональные требования
- Спецификация -> Валидационные правила
- Пользовательские истории -> US-01, US-02

Правило
ID: CR-08
Тип: переход состояния
Формулировка: Статус `завершена` достигается только после выполнения всех шагов либо явного выполнения цели без шагов.
Источник: Лог уточнений: CL-04
Использовать дальше:
- Спецификация -> Бизнес-правила
- Спецификация -> Модель состояний / Переходы
- Пользовательские истории -> US-05, US-06

Правило
ID: CR-09
Тип: сценарий
Формулировка: Пользователь должен видеть статус и процент прогресса каждой активной цели.
Источник: Вход
Использовать дальше:
- Спецификация -> Пользовательские сценарии
- Пользовательские истории -> US-01

Правило
ID: CR-10
Тип: ограничение
Формулировка: В текущем состоянии приложения цель можно только создать или удалить.
Источник: Вход
Использовать дальше:
- Спецификация -> Контекст / Проблема
- Спецификация -> Ограничения
- Отчет о пробелах и рисках -> если потребуется анализ текущего дефицита функциональности
