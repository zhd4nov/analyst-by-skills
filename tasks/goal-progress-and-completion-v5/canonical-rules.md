# Канонические правила

Статус: complete

## CR-01
- Тип: термин / актор
- Формулировка: Основной актор функциональности - пользователь приложения по постановке личных целей.
- Источник: Вход
- Использовать дальше: Спецификация; Пользовательские истории

## CR-02
- Тип: ограничение текущего состояния
- Формулировка: В текущем приложении цель можно создать и удалить, но нельзя учитывать ее прогресс или завершить ее как достижение.
- Источник: Вход
- Использовать дальше: Спецификация; Отчет о пробелах и рисках

## CR-03
- Тип: функциональное правило
- Формулировка: Система должна показывать пользователю прогресс цели, чтобы было понятно, на каком этапе находится цель.
- Источник: Вход
- Использовать дальше: Спецификация; Пользовательские истории

## CR-04
- Тип: расчетное правило
- Формулировка: Если у цели есть шаги, прогресс цели считается в процентах от выполненных шагов.
- Источник: Лог уточнений CL-01
- Использовать дальше: Спецификация; Пользовательские истории

## CR-05
- Тип: расчетное правило
- Формулировка: Если у цели нет шагов, прогресс цели меняется по схеме `0% -> 100%`.
- Источник: Лог уточнений CL-01
- Использовать дальше: Спецификация; Пользовательские истории

## CR-06
- Тип: правило завершения
- Формулировка: Цель может быть завершена при достижении `100%` прогресса.
- Источник: Лог уточнений CL-02
- Использовать дальше: Спецификация; Пользовательские истории

## CR-07
- Тип: правило завершения
- Формулировка: Пользователь может вручную выполнить действие завершения цели.
- Источник: Лог уточнений CL-02
- Использовать дальше: Спецификация; Пользовательские истории

## CR-08
- Тип: ограничение хранения
- Формулировка: Детальную историю изменений прогресса хранить не нужно, но факт достижения цели должен фиксироваться.
- Источник: Лог уточнений CL-03
- Использовать дальше: Спецификация; Отчет о пробелах и рисках

## CR-09
- Тип: правило видимости
- Формулировка: Завершенная цель исключается из активного списка и попадает в список достижений.
- Источник: Лог уточнений CL-04
- Использовать дальше: Спецификация; Пользовательские истории
