# Пользовательские истории

Статус: complete

## История

ID: US-01
Название: Просмотр прогресса активной цели

Источник
- Спецификация -> Пользовательские сценарии -> пользователь просматривает активные цели и видит прогресс каждой цели в процентах
- Спецификация -> Функциональные требования -> система должна показывать прогресс цели в процентах в активном списке целей
- Канонические правила: CR-01, CR-03

Актор: пользователь приложения по постановке личных целей

Потребность: видеть текущий прогресс цели

Ценность: понимать, на каком этапе находится цель и было ли по ней продвижение

Сценарий: пользователь открывает активный список целей или детали цели и видит прогресс цели в процентах.

Критерии приемки
- В активном списке целей у каждой активной цели отображается процент прогресса.
- В деталях цели отображается процент прогресса.
- Для цели без зафиксированного выполнения отображается `0%`.
- Завершенные цели не отображаются в активном списке.

Бизнес-правила
- Прогресс цели должен отображаться в процентах.
- Завершенные цели исключаются из активного списка.

Зависимости
- нет

## История

ID: US-02
Название: Расчет прогресса цели по выполненным шагам

Источник
- Спецификация -> Пользовательские сценарии -> пользователь отмечает шаги цели выполненными, и система пересчитывает прогресс цели
- Спецификация -> Функциональные требования -> если у цели есть шаги, система должна считать прогресс как долю выполненных шагов от общего количества шагов
- Лог уточнений: CL-01
- Канонические правила: CR-04

Актор: пользователь приложения по постановке личных целей

Потребность: отмечать выполненные шаги цели

Ценность: видеть объективный прогресс цели по уже выполненным действиям

Сценарий: пользователь отмечает шаг цели выполненным, после чего система пересчитывает процент прогресса цели.

Критерии приемки
- При выполнении шага цель пересчитывает прогресс на основе количества выполненных шагов и общего количества шагов.
- При изменении состояния шага процент прогресса обновляется.
- При выполнении всех шагов прогресс цели становится `100%`.
- Пользователь не вводит процент прогресса вручную для цели со шагами.

Бизнес-правила
- Прогресс цели со шагами считается в процентах от выполненных шагов.

Зависимости
- наличие шагов у цели

## История

ID: US-03
Название: Выполнение цели без шагов

Источник
- Спецификация -> Пользовательские сценарии -> цель не содержит шагов, поэтому прогресс меняется только между `0%` и `100%`
- Спецификация -> Функциональные требования -> если у цели нет шагов, система должна поддерживать прогресс по схеме `0% -> 100%`
- Лог уточнений: CL-01
- Канонические правила: CR-05

Актор: пользователь приложения по постановке личных целей

Потребность: завершать цель, даже если она не разбита на шаги

Ценность: фиксировать достижение простой цели без необходимости создавать искусственные шаги

Сценарий: пользователь вручную завершает цель без шагов, и прогресс цели меняется с `0%` на `100%`.

Критерии приемки
- Цель без шагов до выполнения имеет прогресс `0%`.
- После ручного выполнения цель без шагов имеет прогресс `100%`.
- Для цели без шагов не используются промежуточные проценты прогресса.

Бизнес-правила
- Для цели без шагов допустим прогресс только по схеме `0% -> 100%`.

Зависимости
- нет

## История

ID: US-04
Название: Завершение цели и перенос в достижения

Источник
- Спецификация -> Пользовательские сценарии -> пользователь завершает цель при достижении `100%` прогресса
- Спецификация -> Пользовательские сценарии -> пользователь вручную завершает цель
- Спецификация -> Функциональные требования -> после завершения цель должна появляться в списке достижений
- Лог уточнений: CL-02, CL-03, CL-04
- Канонические правила: CR-06, CR-07, CR-08, CR-09

Актор: пользователь приложения по постановке личных целей

Потребность: завершать достигнутую цель

Ценность: видеть достигнутые цели отдельно от активных и не смешивать достижение с удалением

Сценарий: цель достигает `100%` прогресса или пользователь вручную завершает цель; система фиксирует факт достижения, убирает цель из активного списка и показывает ее в достижениях.

Критерии приемки
- При завершении цели фиксируется факт достижения.
- Завершенная цель имеет прогресс `100%`.
- Завершенная цель исчезает из активного списка.
- Завершенная цель появляется в списке достижений.
- Детальная история изменений прогресса не создается.

Бизнес-правила
- Цель может быть завершена при достижении `100%` прогресса.
- Пользователь может вручную завершить цель.
- Факт достижения цели должен фиксироваться.
- Завершенная цель переносится в список достижений.

Зависимости
- нет

## История

ID: US-05
Название: Удаление цели без фиксации достижения

Источник
- Спецификация -> Исключительные сценарии -> пользователь удаляет цель: такая цель не должна фиксироваться как достижение
- Спецификация -> Валидационные правила -> удаленная цель не должна попадать в список достижений только из-за удаления
- Канонические правила: CR-02

Актор: пользователь приложения по постановке личных целей

Потребность: удалять цель без отметки достижения

Ценность: отличать отказ от цели от факта ее достижения

Сценарий: пользователь удаляет цель, и система не переносит ее в достижения.

Критерии приемки
- Удаленная цель не отображается в списке достижений.
- Удаление цели не фиксирует факт достижения.
- Завершение и удаление цели являются разными пользовательскими результатами.

Бизнес-правила
- Удаление цели не является завершением цели.

Зависимости
- существующий сценарий удаления цели
