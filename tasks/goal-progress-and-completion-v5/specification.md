# Спецификация

Статус: complete

## Контекст / Проблема

В приложении по постановке личных целей пользователь сейчас может только поставить цель и удалить ее. Из-за этого пользователь не понимает, на каком этапе находится цель, есть ли по ней продвижение и была ли цель действительно достигнута.

## Бизнес-цель

Дать пользователю понятный учет прогресса личных целей и отделить завершение цели как достижение от удаления цели.

## Границы объема

### Внутри объема
- Отображение прогресса цели в процентах.
- Расчет прогресса цели со шагами по выполненным шагам.
- Поддержка целей без шагов по схеме прогресса `0% -> 100%`.
- Завершение цели при достижении `100%` прогресса.
- Ручное действие пользователя для завершения цели.
- Фиксация факта достижения цели.
- Перенос завершенной цели из активного списка в список достижений.

### Вне объема
- Детальная история изменений прогресса.
- Аналитика динамики прогресса по датам.
- Правила удаления цели за пределами отличия удаления от завершения.

## Термины и определения

- Цель: личная цель пользователя в приложении.
- Шаг цели: отдельное действие или этап внутри цели, который может быть выполнен.
- Прогресс цели: процент выполнения цели.
- Активная цель: цель, которая еще не завершена и отображается в активном списке.
- Завершенная цель: цель, по которой зафиксирован факт достижения.
- Достижения: список завершенных целей.
- Удаление цели: удаление цели без фиксации факта достижения.

## Акторы

- Пользователь приложения по постановке личных целей.

## Пользовательские сценарии

### Основные
- Пользователь просматривает активные цели и видит прогресс каждой цели в процентах.
- Пользователь отмечает шаги цели выполненными, и система пересчитывает прогресс цели.
- Пользователь работает с целью без шагов, которая до выполнения имеет прогресс `0%`, а после выполнения - `100%`.
- Пользователь завершает цель при достижении `100%` прогресса.
- Пользователь вручную завершает цель.
- Пользователь открывает список достижений и видит завершенные цели.

### Альтернативные
- Цель не содержит шагов, поэтому прогресс меняется только между `0%` и `100%`.
- Цель завершается ручным действием пользователя, а не только автоматическим достижением `100%`.

### Исключительные
- Пользователь удаляет цель: такая цель не должна фиксироваться как достижение.

## Функциональные требования

- Система должна показывать прогресс цели в процентах в активном списке целей.
- Система должна показывать прогресс цели в процентах в деталях цели.
- Если у цели есть шаги, система должна считать прогресс как долю выполненных шагов от общего количества шагов.
- Если у цели нет шагов, система должна поддерживать прогресс по схеме `0% -> 100%`.
- Система должна позволять пользователю отмечать шаги цели выполненными.
- Система должна пересчитывать прогресс после изменения выполнения шагов.
- Система должна позволять завершить цель при достижении `100%` прогресса.
- Система должна позволять пользователю вручную завершить цель.
- При завершении цели система должна фиксировать факт достижения.
- После завершения цель должна исчезать из активного списка.
- После завершения цель должна появляться в списке достижений.
- Система не должна хранить детальную историю изменений прогресса.
- Система должна отличать завершение цели от удаления цели.

## Валидационные правила

- Для цели со шагами прогресс должен рассчитываться из количества выполненных шагов и общего количества шагов.
- Для цели без шагов допустимы значения прогресса `0%` и `100%`.
- Завершенная цель должна иметь прогресс `100%`.
- Завершенная цель не должна отображаться в активном списке.
- Завершенная цель должна отображаться в списке достижений.
- Удаленная цель не должна попадать в список достижений только из-за удаления.

## Бизнес-правила

- [Источник: Вход] В текущем приложении цель можно поставить и удалить, но нельзя видеть прогресс или завершение.
- [Источник: Лог уточнений CL-01] Прогресс цели со шагами считается в процентах от выполненных шагов.
- [Источник: Лог уточнений CL-01] У цели может не быть шагов; тогда цель может быть выполнена по схеме `0% -> 100%`.
- [Источник: Лог уточнений CL-02] Завершением цели считается достижение `100%` прогресса.
- [Источник: Лог уточнений CL-02] Завершением цели также считается ручное действие пользователя по завершению цели.
- [Источник: Лог уточнений CL-03] Детальную историю изменений прогресса хранить не нужно.
- [Источник: Лог уточнений CL-03] Факт достижения цели должен фиксироваться.
- [Источник: Лог уточнений CL-04] Завершенная цель переносится из активного списка в список достижений.

## Модель состояний / Переходы

- Начальное состояние: активная цель с прогрессом `0%`, если иное не следует из уже выполненных шагов.

### Состояния
- Активная цель.
- Завершенная цель / достижение.
- Удаленная цель.

### Переходы
- Активная цель -> Завершенная цель / достижение
  - Инициатор: система или пользователь.
  - Событие / условие: цель достигла `100%` прогресса или пользователь вручную завершил цель.
  - Источник: Лог уточнений CL-02, CL-04.
- Активная цель -> Удаленная цель
  - Инициатор: пользователь.
  - Событие / условие: пользователь удаляет цель.
  - Источник: Вход.

## Данные / Сущности

- Цель.
- Шаг цели.
- Признак выполнения шага.
- Процент прогресса.
- Факт достижения цели.
- Активный список целей.
- Список достижений.
- Признак удаления цели.

## Ограничения

- [Источник: Лог уточнений CL-03] Детальная история изменений прогресса не входит в объем.
- [Источник: Лог уточнений CL-04] Завершенная цель должна быть исключена из активного списка.
- [Источник: Лог уточнений CL-04] Завершенная цель должна быть доступна в списке достижений.
