# Пользовательские истории

История
ID: US-01
Название: Создание и отправка заявки на командировку
Источник
- Спецификация -> Функциональные требования: создание заявки и отправка руководителю.
- Спецификация -> Данные / Сущности: обязательные данные заявки.
- CR-01
- CR-02
- CR-13
Актор: Сотрудник
Потребность: создать заявку на командировку и отправить ее руководителю на согласование.
Ценность: запустить согласование поездки в едином процессе без переписки в почте или мессенджерах.
Сценарий: сотрудник заполняет даты, город, цель поездки и предварительный бюджет, затем отправляет заявку на согласование руководителю.
Критерии приемки
- Заявка может быть отправлена на согласование только при наличии дат, города, цели поездки и предварительного бюджета.
- После отправки заявка получает статус `на согласовании у руководителя`.
- Отправленная заявка становится доступна руководителю для решения.
- Новая заявка, созданная вместо отмененной, проходит тот же процесс согласования заново.
Бизнес-правила
- После отправки заявка должна попасть руководителю на согласование.
- Новая заявка, созданная вместо отмененной, проходит процесс согласования заново.

История
ID: US-02
Название: Рассмотрение заявки руководителем
Источник
- Спецификация -> Функциональные требования: согласование или отклонение заявки руководителем.
- Спецификация -> Валидационные правила: обязательная причина отклонения.
- CR-03
- CR-04
Актор: Руководитель
Потребность: согласовать или отклонить заявку на командировку.
Ценность: зафиксировать управленческое решение по поездке в едином процессе.
Сценарий: руководитель открывает заявку, принимает решение о согласовании или отклонении; при отклонении указывает причину.
Критерии приемки
- Руководитель может согласовать заявку, находящуюся на согласовании.
- После согласования заявка получает статус `согласована руководителем`.
- Руководитель может отклонить заявку, находящуюся на согласовании.
- При отклонении без причины решение не принимается.
- После отклонения с причиной заявка получает статус `отклонена руководителем`.
Бизнес-правила
- Руководитель может согласовать или отклонить заявку.
- При отклонении причина обязательна.
Зависимости
- нет

История
ID: US-03
Название: Просмотр статуса заявки сотрудником
Источник
- Спецификация -> Функциональные требования: сотрудник видит статус заявки.
- Спецификация -> Модель состояний / Переходы.
- CR-05
Актор: Сотрудник
Потребность: видеть текущий статус своей заявки.
Ценность: понимать, кто уже выполнил действие по заявке и какой этап процесса идет сейчас.
Сценарий: сотрудник открывает свою заявку и видит ее текущий статус.
Критерии приемки
- Для заявки отображается один из статусов, описанных в модели состояний.
- После решения руководителя сотрудник видит статус `согласована руководителем` или `отклонена руководителем`.
- После отметки бухгалтерии сотрудник видит статус `подготовка документов завершена`.
- Для отмененной заявки сотрудник видит статус `отменена`.
Бизнес-правила
- Сотрудник должен видеть статус заявки.

История
ID: US-04
Название: Отмена заявки для изменения данных
Источник
- Спецификация -> Бизнес-правила: после отправки заявку нельзя редактировать.
- Спецификация -> Модель состояний / Переходы: переходы в статус `отменена`.
- Лог уточнений: CL-01
- CR-09
- CR-11
- CR-12
Актор: Сотрудник
Потребность: отменить отправленную заявку, если нужно изменить даты, город, цель поездки или бюджет.
Ценность: изменить данные поездки без незаметной подмены уже отправленной или согласованной заявки.
Сценарий: сотрудник отменяет текущую отправленную заявку и создает новую заявку с актуальными данными.
Критерии приемки
- После отправки сотрудник не может изменить даты, город, цель поездки или бюджет в текущей заявке.
- Сотрудник может отменить отправленную заявку, если требуется изменить даты, город, цель поездки или бюджет.
- После отмены заявка получает статус `отменена`.
- Отмененная заявка больше не участвует в согласовании и подготовке документов.
- Для измененных данных сотрудник создает новую заявку, которая проходит согласование заново.
Бизнес-правила
- После отправки заявку нельзя редактировать.
- Отмененная заявка больше не участвует в согласовании и подготовке документов.

История
ID: US-05
Название: Подготовка документов бухгалтерией
Источник
- Спецификация -> Функциональные требования: доступ бухгалтерии к согласованным заявкам и отметка завершения подготовки.
- Спецификация -> Бизнес-правила: бухгалтерия не согласует поездку.
- CR-06
- CR-07
- CR-08
Актор: Бухгалтерия
Потребность: видеть согласованные заявки и отмечать завершение подготовки аванса и документов.
Ценность: выполнять финансово-документальную подготовку после управленческого согласования без отдельного уточнения статуса в переписке.
Сценарий: бухгалтерия открывает согласованную руководителем заявку, готовит аванс и документы, затем отмечает подготовку завершенной.
Критерии приемки
- Бухгалтерии доступны заявки в статусе `согласована руководителем`.
- Бухгалтерия не может согласовать или отклонить поездку.
- Бухгалтерия может отметить завершение подготовки аванса и документов по согласованной заявке.
- После отметки бухгалтерии заявка получает статус `подготовка документов завершена`.
- Отмененные заявки не участвуют в подготовке документов.
Бизнес-правила
- Бухгалтерия не согласует поездку.
- После согласования руководителем заявка доступна бухгалтерии.
- Бухгалтерия отмечает завершение подготовки аванса и документов.
