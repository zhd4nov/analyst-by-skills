# Уточненные требования

Статус: complete

## Акторы
- Пользователь приложения по постановке личных целей.

## Цели
- Пользователь должен понимать, на каком этапе находится каждая личная цель.
- Пользователь должен видеть прогресс цели в процентах.
- Пользователь должен понимать, сделал ли он что-то для достижения цели.
- Пользователь должен иметь возможность завершить цель, а не только создать или удалить ее.
- Пользователь должен видеть завершенные цели как достижения.

## Текущий функциональный объем
- Добавить учет прогресса личной цели.
- Для цели со шагами считать прогресс в процентах от выполненных шагов. [Лог уточнений: CL-01]
- Для цели без шагов поддержать дискретную схему прогресса `0% -> 100%`. [Лог уточнений: CL-01]
- Добавить завершение цели по достижению `100%` прогресса. [Лог уточнений: CL-02]
- Добавить ручное действие пользователя для завершения цели. [Лог уточнений: CL-02]
- Фиксировать сам факт достижения цели без хранения детальной истории изменений прогресса. [Лог уточнений: CL-03]
- Переносить завершенную цель из активного списка в список достижений. [Лог уточнений: CL-04]

## Ключевые сценарии
- Пользователь просматривает активные цели и видит текущий прогресс каждой цели.
- Пользователь выполняет шаг цели, после чего прогресс цели пересчитывается.
- Пользователь завершает цель при достижении `100%` прогресса.
- Пользователь вручную завершает цель.
- Пользователь открывает список достижений и видит завершенные цели.

## Известные ограничения
- Текущая система позволяет только поставить цель и удалить цель.
- Детальную историю изменения прогресса хранить не нужно. [Лог уточнений: CL-03]
- Завершение цели должно отличаться от удаления: завершенная цель становится достижением, а не просто исчезает. [Лог уточнений: CL-04]
