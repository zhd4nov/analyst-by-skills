# Отчет о пробелах и рисках

Статус: complete

## Пробел
ID: GAP-01
Статус пробела: закрыт
Источник пробела
- Спецификация -> Пользовательские сценарии -> Исключительные
- Пользовательская история US-01
- Лог уточнений: CL-02
- Лог уточнений: CL-05
Область: исключительный сценарий определения руководителя
Описание пробела: было не определено поведение системы, если в оргструктуре для сотрудника не найден руководитель.
Почему это важно: основной поток зависит от автоматического определения руководителя, а ручной выбор руководителя сотрудником исключен.
Риск: средний
Влияние: для первой версии не влияет на основной процесс, потому что scope ограничен сотрудниками с определенным руководителем.
Что нужно уточнить: ничего для текущего scope; поведение для отсутствующего руководителя нужно определить перед расширением scope.
Предлагаемое решение: оставить исключительный кейс вне первой версии и не включать его в командный scope текущего запуска.
Блокирует разработку: нет

## Пробел
ID: GAP-02
Статус пробела: закрыт
Источник пробела
- Спецификация -> Модель состояний / Переходы -> `Отменена`
- Пользовательская история US-03
- Лог уточнений: CL-04
- Лог уточнений: CL-06
Область: модель состояний заявки
Описание пробела: статус `Отменена` подтвержден, но действие отмены и переходы в этот статус исключены из первой версии.
Почему это важно: без правил перехода невозможно полноценно проверить поведение этого статуса и права на действие отмены.
Риск: средний
Влияние: для первой версии не влияет на основной процесс, потому что действие отмены не входит в scope.
Что нужно уточнить: ничего для текущего scope; правила отмены нужно определить перед разработкой действия отмены.
Предлагаемое решение: оставить действие отмены и переходы в `Отменена` вне первой версии; в текущем scope использовать статус только как зарезервированный.
Блокирует разработку: нет
