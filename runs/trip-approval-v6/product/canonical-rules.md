# Канонические правила

Статус: complete

## Правило
ID: CR-01
Тип: данные
Формулировка: заявка на командировку должна содержать даты, город, цель поездки и предварительный бюджет.
Источник: Вход
Использовать дальше:
- Спецификация -> Данные / Сущности
- Пользовательские истории

## Правило
ID: CR-02
Тип: сценарий
Формулировка: после отправки заявка должна попасть руководителю на согласование.
Источник: Вход
Использовать дальше:
- Спецификация -> Пользовательские сценарии
- Пользовательские истории

ID: CR-03
Тип: бизнес-правило

## Правило
ID: CR-11
Тип: источник данных
Формулировка: руководитель для согласования подтягивается из оргструктуры, сотрудник не выбирает руководителя вручную.
Источник: Лог уточнений: CL-02
Использовать дальше:
- Спецификация -> Функциональные требования
- Пользовательские истории

## Правило
ID: CR-12
Тип: граница объема
Формулировка: если руководитель не определен в оргструктуре, этот исключительный кейс не входит в scope первой версии.
Источник: Лог уточнений: CL-05
Использовать дальше:
- Спецификация -> Границы объема

## Правило
ID: CR-13
Тип: состояния
Формулировка: подтвержденный набор статусов заявки: `Черновик`, `На согласовании`, `Отклонена`, `Согласована`, `Документы готовятся`, `Документы готовы`, `Отменена`; статус `Отменена` является допустимым будущим/зарезервированным статусом, а действие отмены и переходы в него не входят в первую версию.
Источник: Лог уточнений: CL-04, Лог уточнений: CL-06
Использовать дальше:
- Спецификация -> Модель состояний / Переходы
- Пользовательские истории
