# Пользовательские истории

## История

ID: US-01
Название: Подать заявку на командировку
Источник:
- Спецификация -> Функциональные требования: создание заявки и указание дат, города, цели поездки и предварительного бюджета.
- Спецификация -> Функциональные требования: редактирование дат или бюджета в отправленной заявке с повторным согласованием.
- Спецификация -> Валидационные правила: обязательность дат, города, цели и предварительного бюджета.
- CR-01, CR-02, CR-03, CR-04, CR-05, CR-13, CR-14.
Актор: сотрудник.
Потребность: создать и отправить заявку на командировку с ключевыми параметрами поездки.
Ценность: заявка попадает в управляемый процесс согласования без переписки в почте.
Сценарий: сотрудник заполняет заявку датами, городом, целью поездки и предварительным бюджетом, затем отправляет ее руководителю.
Критерии приемки:
- Сотрудник может создать заявку с датами, городом, целью поездки и предварительным бюджетом.
- Заявку нельзя отправить без дат поездки.
- Заявку нельзя отправить без города поездки.
- Заявку нельзя отправить без цели поездки.
- Заявку нельзя отправить без предварительного бюджета.
- После отправки заявка переходит в статус "На согласовании у руководителя".
- Если сотрудник меняет даты или бюджет после отправки, заявка повторно уходит руководителю на согласование.
- Изменение отправленной заявки не должно происходить незаметно для процесса согласования.
Бизнес-правила:
- После отправки сотрудник не должен иметь возможность незаметно изменить заявку.
- Изменение дат или бюджета после отправки требует повторного согласования руководителем.
Зависимости:
- нет.
Открытые вопросы:
- нет.

## История

ID: US-02
Название: Согласовать или отклонить заявку
Источник:
- Спецификация -> Функциональные требования: согласование или отклонение заявки руководителем.
- Спецификация -> Валидационные правила: обязательная причина при отклонении.
- CR-06, CR-07, CR-08.
Актор: руководитель.
Потребность: принять решение по заявке сотрудника.
Ценность: участники процесса видят зафиксированное решение вместо разрозненной переписки.
Сценарий: руководитель рассматривает заявку и выбирает согласование или отклонение; при отклонении указывает причину.
Критерии приемки:
- Руководитель может согласовать заявку, находящуюся на согласовании.
- После согласования заявка получает статус "Согласована руководителем".
- Руководитель может отклонить заявку, находящуюся на согласовании.
- Отклонение невозможно без причины.
- После отклонения заявка получает статус "Отклонена руководителем".
Бизнес-правила:
- Согласование поездки выполняет руководитель.
Зависимости:
- Есть отправленная заявка в статусе "На согласовании у руководителя".
Открытые вопросы:
- нет.

## История

ID: US-03
Название: Видеть статус заявки
Источник:
- Спецификация -> Функциональные требования: сотрудник должен видеть статус заявки.
- Спецификация -> Модель состояний / Переходы.
- CR-09.
Актор: сотрудник.
Потребность: видеть текущее состояние своей заявки.
Ценность: сотрудник понимает, согласована ли поездка, отклонена ли она или документы еще готовятся.
Сценарий: сотрудник открывает свою заявку и видит ее текущий статус.
Критерии приемки:
- Сотрудник видит статус отправленной заявки.
- Статус отражает один из зафиксированных этапов: "На согласовании у руководителя", "Согласована руководителем", "Отклонена руководителем", "Документы подготовлены".
- Для отклоненной заявки сотруднику доступна причина отклонения.
Бизнес-правила:
- Статус должен отражать зафиксированное действие участника процесса.
Зависимости:
- Есть заявка сотрудника.
Открытые вопросы:
- нет.

## История

ID: US-04
Название: Подготовить документы по согласованной заявке
Источник:
- Спецификация -> Функциональные требования: доступ бухгалтерии к согласованным заявкам и отметка завершения подготовки.
- Спецификация -> Бизнес-правила: бухгалтерия не согласует саму поездку.
- CR-10, CR-11, CR-12.
Актор: бухгалтерия.
Потребность: видеть согласованные заявки и отмечать завершение подготовки аванса и документов.
Ценность: подготовка документов становится видимой частью процесса без отдельного уточнения в переписке.
Сценарий: бухгалтерия получает доступ к согласованной заявке, готовит аванс и документы, затем отмечает подготовку завершенной.
Критерии приемки:
- Бухгалтерия видит заявку после ее согласования руководителем.
- Бухгалтерия не выполняет действие согласования поездки.
- Бухгалтерия может отметить, что подготовка аванса и документов завершена.
- После отметки бухгалтерии заявка получает статус "Документы подготовлены".
Бизнес-правила:
- Бухгалтерия работает только с согласованными руководителем заявками.
- Бухгалтерия не согласует саму поездку.
Зависимости:
- Заявка согласована руководителем.
Открытые вопросы:
- нет.
