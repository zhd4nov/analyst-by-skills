# Канонические правила

Правило
ID: CR-01
Тип: данные
Формулировка: Заявка на командировку содержит даты, город, цель поездки и предварительный бюджет.
Источник: Вход
Использовать дальше:
- Спецификация -> Данные / Сущности
- Пользовательские истории -> создание заявки

Правило
ID: CR-02
Тип: переход состояния
Формулировка: После отправки заявка переходит руководителю на согласование.
Источник: Вход
Использовать дальше:
- Спецификация -> Модель состояний / Переходы
- Пользовательские истории -> отправка заявки на согласование

Правило
ID: CR-03
Тип: бизнес-правило
Формулировка: Руководитель может согласовать или отклонить заявку.
Источник: Вход
Использовать дальше:
- Спецификация -> Бизнес-правила
- Пользовательские истории -> согласование заявки руководителем

Правило
ID: CR-04
Тип: валидация
Формулировка: При отклонении заявки руководитель обязан указать причину.
Источник: Вход
Использовать дальше:
- Спецификация -> Валидационные правила
- Пользовательские истории -> отклонение заявки руководителем

Правило
ID: CR-05
Тип: видимость
Формулировка: Сотрудник должен видеть статус своей заявки.
Источник: Вход
Использовать дальше:
- Спецификация -> Функциональные требования
- Пользовательские истории -> просмотр статуса заявки

Правило
ID: CR-06
Тип: видимость
Формулировка: После согласования руководителем заявка должна быть доступна бухгалтерии.
Источник: Вход
Использовать дальше:
- Спецификация -> Функциональные требования
- Пользовательские истории -> работа бухгалтерии с согласованными заявками

Правило
ID: CR-07
Тип: бизнес-правило
Формулировка: Бухгалтерия не согласует поездку.
Источник: Вход
Использовать дальше:
- Спецификация -> Бизнес-правила
- Пользовательские истории -> работа бухгалтерии с согласованными заявками

Правило
ID: CR-08
Тип: переход состояния
Формулировка: Бухгалтерия может отметить завершение подготовки документов по согласованной заявке.
Источник: Вход
Использовать дальше:
- Спецификация -> Модель состояний / Переходы
- Пользовательские истории -> завершение подготовки документов

Правило
ID: CR-09
Тип: ограничение
Формулировка: В первой версии после отправки заявку нельзя редактировать.
Источник: Лог уточнений: CL-01
Использовать дальше:
- Спецификация -> Ограничения
- Пользовательские истории -> отмена заявки вместо редактирования

Правило
ID: CR-10
Тип: переход состояния
Формулировка: Если сотруднику нужно изменить даты, город, цель или бюджет после отправки, он отменяет текущую заявку и создает новую.
Источник: Лог уточнений: CL-01
Использовать дальше:
- Спецификация -> Модель состояний / Переходы
- Пользовательские истории -> отмена заявки вместо редактирования

Правило
ID: CR-11
Тип: бизнес-правило
Формулировка: Отмененная заявка больше не участвует в согласовании и подготовке документов.
Источник: Лог уточнений: CL-01
Использовать дальше:
- Спецификация -> Бизнес-правила
- Пользовательские истории -> отмена заявки вместо редактирования

Правило
ID: CR-12
Тип: ограничение
Формулировка: В первой версии вне объема бронирование билетов, интеграция с календарем, расчет суточных и хранение чеков.
Источник: Вход
Использовать дальше:
- Спецификация -> Границы объема
