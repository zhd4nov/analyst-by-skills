# Шаблон: Канонические правила

Использовать как операциональный слой между `Уточненными требованиями` и `Спецификацией`.

```text
Канонические правила

Правило
ID: CR-01
Тип: <термин / сценарий / бизнес-правило / валидация / переход состояния / ограничение / данные / видимость>
Формулировка: <одно атомарное утверждение без пересказа и без смешения нескольких правил>
Источник: <Вход / Лог уточнений: CL-01 / Допущение>
Использовать дальше:
- <Спецификация -> конкретный раздел>
- <Пользовательские истории -> если правило влияет на истории>
- <Отчет о пробелах и рисках -> если отсутствие детализации по правилу создает риск>

Правило
ID: CR-02
Тип: <термин / сценарий / бизнес-правило / валидация / переход состояния / ограничение / данные / видимость>
Формулировка: <одно атомарное утверждение без пересказа и без смешения нескольких правил>
Источник: <Вход / Лог уточнений: CL-02 / Допущение>
Использовать дальше:
- <Спецификация -> конкретный раздел>
```

## Правила использования

- Один пункт содержит только одно правило.
- `Формулировка` должна быть короткой и пригодной для прямой вставки в downstream-артефакты.
- Если факт отсутствовал во `Входе`, но получен из ответа пользователя, `Источник` должен ссылаться на запись в `Логе уточнений`.
- Если правило не влияет ни на один downstream-артефакт, его не нужно включать.
- Если несколько правил относятся к одному сценарию, их нужно фиксировать отдельными пунктами, а не длинным абзацем.
- Если правило описывает поведение, которое меняет приемку, его нужно пометить так, чтобы оно попало в `Спецификацию` и при необходимости в `Пользовательские истории`.
- Если факт пока не подтвержден, он не должен попадать в `Канонические правила`; его нужно оставить в `Допущения` или `Открытые вопросы`.
