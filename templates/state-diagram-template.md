# Шаблон: Диаграмма состояний

Использовать только как побочный артефакт этапа `spec-structurer`, если в требованиях есть статусы, состояния, этапы жизненного цикла или правила переходов и человеку полезна визуализация.

```text
Диаграмма состояний

- Начальное состояние: <если известно>
- Состояния:
  - <Состояние_1>
  - <Состояние_2>
  - <Состояние_3>
- Переходы:
  - <Состояние_1> -> <Состояние_2>: <событие / условие перехода>
  - <Состояние_2> -> <Состояние_3>: <событие / условие перехода>
  - <Состояние_3> -> <Конечное_состояние>: <событие / условие перехода>
```

## Правила использования

- Это вспомогательный артефакт для визуализации поведения, а не замена `Спецификации`.
- Downstream-агенты не должны зависеть от этого артефакта и не должны нуждаться в нем для продолжения пайплайна.
- Каждое состояние и каждый переход должны иметь опору в текстовом разделе `Модель состояний / Переходы` внутри `Спецификации`.
- Если правило перехода не определено, его нельзя изображать как подтвержденный переход.
- Имена состояний должны быть однозначными и соответствовать терминологии спецификации.
- Если начальное состояние неизвестно, это должно попадать в `Открытые вопросы`, а не маскироваться произвольным выбором.
