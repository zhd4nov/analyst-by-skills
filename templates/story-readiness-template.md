# Шаблон: Статус готовности историй

Использовать как минимально достаточный результат после этапа проверки пользовательских историй.

```text
Статус готовности историй

## US-01
Статус готовности: <Высокий / Средний / Низкий>

Проверки:
- <что проверено и почему статус допустим>
- <что проверено и почему статус допустим>

Дефекты:
- <дефект, если есть>

Связанные GAP:
- <GAP-xx, если история связана с пробелом>

## US-02
Статус готовности: <Высокий / Средний / Низкий>

Проверки:
- <что проверено и почему статус допустим>

Дефекты:
- нет

Связанные GAP:
- нет

Открытые вопросы
- <вопрос, если он влияет на готовность истории>
```

## Правила использования

- Каждая история должна иметь отдельный блок `## US-xx`.
- В каждом блоке должна быть строка `Статус готовности: Высокий`, `Статус готовности: Средний` или `Статус готовности: Низкий`.
- `Высокий` допустим только если история имеет источник, актора, потребность, ценность, проверяемые критерии приемки и не связана с блокирующим GAP.
- Если история связана с GAP `Блокирует разработку: да`, статус должен быть не выше `Средний`, пока GAP не закрыт.
- Если GAP делает невозможным определить границы, источник, ценность или приемку истории, статус должен быть `Низкий`.
- Если история не связана с GAP, ее статус не должен снижаться только из-за наличия GAP в других частях отчета.
- `Дефекты` и `Связанные GAP` можно указывать как `нет`, если их нет.
- `Открытые вопросы` должны содержать только вопросы, влияющие на готовность историй. Если вопросов нет, писать `нет`.
